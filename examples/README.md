# Example Files

These files are lightweight examples extracted from the current repository.

They are intended to help reviewers inspect:

- the original source input
- the translated version
- the back-translated version
- the model response
- the doctor-review and modification-score artifacts used for real-world evaluation when available

Some examples are normalized for inspection:

- `healthbench/zh_back_translation_case.json` uses the reviewer script field name `back_translation`.
- `realworld/english_summary_case.json` and `realworld/english_back_to_chinese_summary_case.json` keep the original `medical_record` and add `predicted_medical_record`, even though the production summary files in the full repository store the model output inside `medical_record`.
- `realworld/english_summary_doctor_review_case.json` and `realworld/english_back_to_chinese_doctor_review_case.json` come directly from the real exported doctor-review JSON files used in the full repository workflow.

The examples are generated by:

```bash
python scripts/build_examples_from_repo.py --repo-root .
```
